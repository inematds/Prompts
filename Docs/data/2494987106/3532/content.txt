"Aja como um engenheiro de prompts de nível mundial com especialização em Vibe Coding para Lovable.  
Crie estes **3 prompts** para o Lovable.dev construir:

**Contexto do App**  
Estamos criando o "AI Offer Optimizer" – uma ferramenta onde donos de negócios:  
1. Descrevem seus negócios e ofertas atuais para clientes  
2. Recebem estratégias geradas por IA para aprimorar essas ofertas com automação/IA  
3. Visualizam sugestões refinadas que podem ser implementadas imediatamente  

**Prompt 1: Estrutura Base (Foundational Shell)**  
- Defina o fluxo do usuário: Formulário de entrada → Carregando → Resultados  
- Especifique os campos obrigatórios: Nome do negócio, setor, detalhes da oferta  
- Funcionalidades obrigatórias: Processamento assíncrono, validação de entrada  

**Prompt 2: Integração com Supabase + Webhook**  
- Suponha que o Supabase esteja conectado  
- Instruções:  
  - Crie uma função edge que envie os dados para WEBHOOK_URL_PLACEHOLDER  
  - Valide a estrutura dos dados: `{ businessName: string, industry: string, offer: string }`  
  - Armazene as respostas da IA na tabela `user_offers`  

**Prompt 3: Aperfeiçoamento da Interface (UI Polish)**  
- Requisitos visuais:  
  - Estética profissional de negócios (Tailwind)  
  - Indicador de progresso animado durante o processamento da IA  
  - Layout em cartões para os resultados com botões de copiar para a área de transferência  

Crie também o fluxo de trabalho no n8n que complementará este app do Lovable  

**Requisitos do JSON do Fluxo n8n**  
- 3 nós do GPT-4o:  
  1. "Sugira melhorias com IA para {{$json.businessName}}'s {{$json.offer}} no setor {{$json.industry}}"  
  2. "Converta {{$json.AI_Analysis}} em etapas acionáveis"  
  3. "Formate {{$json.AI_Review}} como marcadores em markdown"  
- Nós de gatilho/resposta via webhook  
- Sintaxe compatível com n8n v1.91.0+
